## Types of Contributions

We welcome contributions in the following areas:

### Academic/Theoretical Contributions
- **New domains**: Identify additional domains exhibiting the Formatic mark
- **Proofs**: Extend or refine the necessity theorems
- **Implementations**: Demonstrate the mark in other programming languages
- **Categorical framework**: Extend the category-theoretic foundations

### Code Contributions
- **Bug fixes**: Corrections to implementation errors
- **Tests**: Additional test cases for existing functionality
- **Documentation**: Improvements to code comments and docstrings
- **Examples**: Educational demonstrations of Formatics concepts

### What We Do NOT Accept
- **Feature requests** that diverge from core theory
- **Architectural changes** without prior discussion and approval
- **Dependencies** that introduce licensing conflicts or bloat
- **Style-only changes** without substantive improvement
- **AI-generated content** without human review and understanding

## Contribution Process

1. **Discuss first**: Open an issue to discuss your proposed contribution before investing significant effort.

2. **Fork and branch**: Create a fork and work on a descriptive branch (e.g., `fix/anchor-validation`, `docs/categorical-interpretation`).

3. **Follow guidelines**:
   - Read [CONTRIBUTING-AGENT.md](CONTRIBUTING-AGENT.md) for code style and structure
   - Read [FORMATICS_AGENT_PROTOCOL.md](FORMATICS_AGENT_PROTOCOL.md) for quick reference
   - Maintain consistency with existing code style
   - Add tests for new functionality
   - Update documentation as needed

4. **Submit a pull request**:
   - Clearly describe what the contribution does and why
   - Reference any related issues
   - Ensure all tests pass
   - Agree to the CLA terms (implied by submitting the PR)

5. **Review process**:
   - The copyright holder will review all contributions
   - Changes may be requested before acceptance
   - The copyright holder has sole discretion to accept or reject contributions

## Academic vs. Commercial Use

### Academic Contributors
If you are contributing from an academic institution:
- Your contribution falls under the academic-use exemption in the LICENSE
- You may cite Formatics in your academic work (see [CITATION.cff](CITATION.cff))
- You must still assign ownership of your contribution as described above

### Commercial/Applied Contributors
If you are contributing from a commercial entity or for applied research:
- You **must obtain written permission** from the copyright holder before contributing
- Standard contributions are presumed to be for academic purposes only
- Commercial use of Formatics (including contributions) requires explicit licensing

## Anti-Hijacking Provisions

**Formatics is a discovered mathematical structure, not an invention.**

Contributors agree:
- Not to claim independent discovery or invention of the Formatic mark
- Not to create derivative frameworks that obscure attribution to Formatics
- Not to delink theoretical concepts from their origin in this repository
- Not to use contributions to build competing proprietary or commercial products

Violations of these provisions constitute breach of the CLA and may result in:
- Revocation of contribution privileges
- Legal action for copyright infringement
- Public notice of violation
